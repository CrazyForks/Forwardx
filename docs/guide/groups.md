# 转发组、入口组和出口组

这三类资源用途不同：转发组负责业务入口故障转移，入口组和出口组负责复用主机集合。

## 转发组

转发组让多台入口主机为同一个目标提供服务，并可通过 DDNS 切换当前入口。

```text
入口 A -> 目标
入口 B -> 目标
入口 C -> 目标
```

成员顺序表示优先级。当前入口失效并达到组内配置的故障窗口后，系统切换到下一可用成员。

- 开启「恢复后切回」：高优先级成员恢复后切回。
- 关闭「恢复后切回」：保持当前成员，直到它再次失败。
- 转发组仅使用主机成员；隧道多入口请使用入口组。
- 业务端口、协议和目标地址仍在「转发规则」中配置。

## 入口组

入口组把多台入口主机保存为一个可复用资源，可供隧道或转发链引用。入口组可以配置统一 DDNS 域名，也可以关闭自动解析后自行维护记录。

引用入口组后，系统按成员生成对应入口路径；切换成员或地址后，会同步刷新所有引用资源。

## 出口组

出口组把多个出口主机保存为一个隧道出口资源，支持出口复用、故障转移和多出口负载均衡。

规则列表显示出口组名称，并可列出实际使用的出口节点；不会再用第一台主机名称替代出口组名称。

## 状态与排序

- “在线”只表示成员 Agent 在线。
- “已启用”表示组配置已打开。
- “可用”由组内成员和最近探测结果决定。
- 拖动成员可调整优先级，保存后会刷新引用关系。

名称应能直接区分用途，例如“香港入口”“日本出口”和“游戏入口故障转移”。
