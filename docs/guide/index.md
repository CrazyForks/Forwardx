# 教程概览

ForwardX 用于管理多台 Linux 服务器上的转发资源、业务规则、用户权限、流量和延迟数据。

第一次部署建议按顺序阅读：

1. [快速开始](./quick-start.md)
2. [部署前准备](./preparation.md)
3. [部署面板](./deploy-panel.md)
4. [环境变量](./env-vars.md)
5. [首次初始化](./first-setup.md)
6. [安装 Agent](./agent.md)
7. [创建转发规则](./rules.md)

## 基本流程

```text
部署面板 -> 初始化数据库 -> 创建 Agent Token -> 安装 Agent
-> 创建链路资源 -> 添加转发规则 -> 检查运行状态和链路测试
```

## 常用页面

| 页面 | 用途 |
| --- | --- |
| 主机管理 | 查看 Agent 在线状态、资源占用、地址和版本 |
| 链路管理 | 管理端口转发、隧道、转发链、转发组、入口组和出口组 |
| 转发规则 | 配置用户访问端口、协议、目标地址和所属资源 |
| 服务管理 | 管理独立延迟探测服务和图表 |
| 用户管理 | 管理用户、权限、额度和到期时间 |
| 系统设置 | 配置公开地址、通知、升级和备份 |
| 插件管理 | 安装插件并管理插件权限、Agent 同步和动态资源 |
